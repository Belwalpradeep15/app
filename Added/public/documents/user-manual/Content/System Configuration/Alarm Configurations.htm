Alarm Configurations

The alarm configuration dialogue is accessed from the Admin Menu.

Explanations

For this feature... Do this...
Enable Alarms Select whether alarms are enabled.
Counting Period Type the frequency in seconds that Panoptes runs the algorithm wherein it groups all events, and also groups all events with a specified Counting Width.
Event aggregation enabled Events are stored and reported in groups rather than individually to reduce storage and communication overheads.
Retrigger alarms & Time to retrigger Where an alarm has been reported but not acknowledged, this feature can be used to return the alarm to prominence after a defined time.
Require Acknowledgement Whether an alarm requires that a Panoptes user manually select the Acknowledgement check box before an alarm can be dismissed.
Clear | Green | Amber | Red Alarms

Select what appears on the portal for a given alert levels (where green is infrequent and red is frequent) as follows: 

  • Icons - an icon marker for the given type of alarm appears on the fibre diagram.
  • Icons and List - as above, with the added condition that the alarm appears in the Alarms dialog list.
  • Icons, List and dialog s - as above, with the added condition that the alarm appears in the Alarms dialog.

The criteria upon which an event or series of events is designated as a green, amber, or red alert is specified using the Event Alarms pane below.
System Alarms These are alarms of which users are always notified.
Play Sound | Loop Sound

Select the type of alarm to play when a system component fails or an event occurs. Further select whether the sound is a one time event or is repeated.
Event Alarms Click New Alarm Configuration () to display the New Alarm Configurations dialog where you select the event type for which to set alarm level criteria. Default criteria populate the new column. Click Template () below a given column to apply different criteria to that column. These are default templates that Fotech has developed to match well to real world scenarios.
General Configuration  
Update Position On Specify whether to mark the alarm position as the location where either the first or the latest event is detected. Last Event is ideal for Given a moving event, such as walking or a train. First Event is ideal for an event that occurs at a fixed location, such as a leak.
Play Sound | Loop Sound Specify whether a warning sound is played and if so, repeated until the alarm condition is resolved.
Threat Configuration  
Is Active

This option has two functions:

  • Whether the alarm profile is used to aggregate matching events as they occur.
  • Whether the icon for an event type, which is aggregated by this alarm appears on the map or schematic. If this option is cleared, the Unknown Event Type appears instead of a specific event type icon.
Data logging Enables data logging associated with the specific alarm type.
Counting Width If an event occurs within half of this distance of the current alarm position, add that event to the current alarm queue of that alarm type so that it affects the overall event count -even if it's not of the same event type.
Decrement

Reduce the count by this number of events when no further events occur within the time period specified in the Counting Width above. Where the number of events for an alarm that has reached a clear, green, or amber threshold are reduced so that the number of events fall below the number of events specified for that threshold, remove that alarm status. Note that once a red alarm condition is reached, it cannot be decremented. It musts be cleared by a Panoptes users.

The alarm condition does not descend from one alarm status down to the next status, the alarm is simply closed and cleared.

This should be a negative value. I find this idea (a double-negative decrement-minus) confusing. Should it be signed?
Green | Amber | Red Threshold The number of events that signifies whether a series of events is considered green, amber, or red alerts. Where the number of events in the event pool meet or exceed the number specified here for a given threshold, escalate the alarm status to the next level.
Hold Count Threshold Where the number of events collected since the last Counting Period matches this value, neither increment nor decrement the number of events. Keep the current number of events in the event pool the same.
Low | Medium | Top Count Threshold Where the number of events collected since the last Counting Period matches or exceeds one of these values, add the value to the event pool specified as the Low, Medium, or Top Count Increment.
Low | Medium | Top Count Increment Add this number to the event pool when the number of events collected since the last Counting Period matches or exceeds the value specified for the Low, Medium, or Top Count Threshold.
Amber Hysteresis When the number of events collected since the last Counting Period passes the Amber threshold, the number specified here is added to the event pool count. This is used to apply velocity to the escalation, whereby when a lot of events occur in rapid succession, the Red Alarm level can quickly be reached.
Amber Clearance When the number of events collected since the last Counting Period falls short of the Amber threshold, the number specified here is removed from the event pool count. This is used to remove velocity from the escalation.
Always red after... Enabling this feature means that any threat which persists for the defined number of seconds will immediately be escalated to a Red threat level.
Initial threat level A threat can be configured to always start at a defined threat level rather than at clear (default level). This allows certain alarms to escalate more rapidly or to generate alarms immediately.
