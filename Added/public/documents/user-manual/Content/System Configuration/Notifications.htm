Notifications

The notifications dialogue is accessed from the Admin Menu.

Explanation

This Menu Item... Does this...
General Notification Settings

Specifies the time zone origin of the information Panoptes outputs, the geospatial format, and language.
XML Notifications

Specify the complete URL (including the "http://" prefix and the port number unless it is the default ":80/") where Panoptes pushes XML data.

Specify multiple machines by separating each URL with a semi-colon. white space characters can be added on either side of the semi-colon to improve readability.

Select the type of data pushed to the above URL (Alarms, Heartbeats, System Health, etc.)

Each event message includes at least Type, Time, Details, Tags, and Location data. Where routes are defined, those details are included as well. Route information enables the direction and relative distance from real world location. For repeat events, it can further include calculation of velocity and acceleration data.

To select the type of events to push, click ... and toggle the desired events by clicking the check boxes to the right of a given event.
Email / SMS Notifications

Click Add () to display the Email/SMS Notification dialog , where you toggle whether the notification is Active. For example, you might set up a vacation schedule that can later be disabled instead of deleted.

Specify a descriptive name for the notification to configure. Type the email address of the recipient(s) email, or if using SMS, type the email address of an email-to-SMS conversion service or your own email2sms gateway. Type the days and times when to send notifications to this recipient. In this way, it's possible to create several notifications, each addressed to different recipients that aligns with their work schedule.

Specify multiple recipients by separating each with a semi-colon. White space characters can be added on either side of the semi-colon to improve readability.

Select the alarm levels at which to notify recipients. System alarms, which are of a more technical nature, can be sent specifically to systems personnel to investigate.

Event-based alarm notifications can be filtered based on the associated threat level, so that certain users only receive email notification when alerts are red, and others receive notification when alerts reach amber.

Enter a subject line; otherwise, Panoptes inserts a generic subject line.
Send Test Email Enter a recipient email address and then click Send Test Email to verify that email functionality is working.
