Configuring Notification Pushes

When an event is received by Panoptes from the Helios unit, Panoptes stores the event, but also forwards the event in XML mark-up to a Uniform Resource Locator (URL) that you specify. Software on the receiving end can post the incoming events to a source you specify.

Configure the event submission using the XML Notifications pane in the Admin Menu > Notifications window. Click ... to specify the type of events to submit.

Route details including the direction of travel and relative distance of an event from a real world location, and --for repeat events, velocity and acceleration data are supported by the XML event push.

Notification can consist of email or SMS messages to different email addresses depending on whether a general alarm (based on a threshold volume for a given type of event being exceeded) or a system alarm occurs. For SMS messages, after subscribing to an email-to-SMS conversion service, or setting up your own email2sms gateway, specify the mobile device number followed by @email2sms.com where email2sms is replaced by the specific subscription or server email address.

To configure notification pushes

Custom email notification groups can be created, each of which can include one or more recipients, and a date and time schedule can be applied to account for work schedules. Email notifications can further define whether event based alarms or system (technical) alarms are sent. If event based alarms, further refinement can be applied so that only alarms that reach a certain threat level are sent.

  1. From the Admin menu, click Notifications. The Notifications dialog appears.
  2. Specify the details for notifications (events, alarms, etc.) sent from the Panoptes unit to a specific URL or email address.
  3. Click Submit.

Email notifications can be altered by clicking Edit (). They can be disabled (instead of deleted), by clearing the Active check box that appears at the top of the Email Notification dialog.

To configure notification processing

Configure how incoming notifications received from the Panoptes unit are processed by the receiving computer. The receiving machine must be able to accept XML messages over HTTP.

XML messages include the system identification details added using the Admin > System Identity window.

For details about processing XML notifications using the Helios unit, refer to the Helios help. Specifically, the Streaming and Reporting Data topic.
