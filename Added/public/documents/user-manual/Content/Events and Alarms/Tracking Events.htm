Tracking Events

Multiple events caused by the same disturbance are all displayed on the map or schematic, which can clutter the display. Events that occur within a certain distance and time period can be grouped as a set of related events so that only one marker appears on the display.

This event relationship is maintained so that events are displayed as groups when you query historical events.

Event tracking must be enabled by your administrator.

To track events

  1. With the Recent Events dialog displayed, click an event. The Event at dialog appears showing details. An event marker appears at the location to which the event you selected pertains and this event marker is connected to the Event dialog with a line callout.
  2. Click Show Track in the bottom right of the dialog. The Event Track dialog appears displaying all of the events related to a track. If the event you clicked is from a historical search, all of the related events —even if they were not returned by your original historical search— appear in the Event Track dialog.
  3. Using the Event Track dialog , click an event, and then click Play() to show the events in chronological sequence.
    The event marker moves around the highlighted area on the fibre line to show the specific location where each event was detected. The frequency with which the different events are shown is fixed and does not represent the actual frequency with which the events occurred.
