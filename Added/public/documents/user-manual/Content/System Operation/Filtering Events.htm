Filtering Events

View a subset of your data by filtering which live events appear in the Recent Events dialog on your specific browser client. Events can be filtered by fibre line, a portion of the fibre line (if using a map display), by event signature, and by signal magnitude, width, and speed.

Panoptes event filtering only affects the events which are reported to the user, it does not affect the events being received from the Helios and being written to the Panoptes database. Filtering the events at source occurs during the commissioning of the Helios unit.

A message appears at the bottom-right of the dialog notifying you that filtering is applied and that you are not receiving all of the possible events. A link labelled Clear Filter to the right of the notification enables you to disable filtering.


To filter events

  1. From the Events menu, select Filter. To filter so that only events that occur in a portion of the fibre (spatial region) on a map display, click and select a box to define the spatial region from which to show events. To clear this selection, click . The Event Filter dialog appears.
  1. Select whether to filter events so that those that appear are as follows:
    • Occur along a specific fibre line or spatial area of that specific fibre.
    • Match a specific pre-defined event signature.
    • Fall within a given magnitude, width, and speed that you define.

Filter criteria are joined with an AND operand so that an event must be true for all conditions, not just one condition, in order for it to appear.

  1. Click Submit.
